# FASE 2: MEDIR (Measure)

## Objetivo

Recopilar datos cuantitativos del proceso actual, establecer la línea base de performance y entender la variación inherente del proceso.

---

## 📄 Archivos en Esta Fase

### 01-RECOLECTA_DATOS.xlsx
**Plan de Recolecta y Datos Base**

Documenta:
- Qué se mide (defect rate, dimensiones, tiempo)
- Cómo se recopilan los datos
- Frecuencia de recolecta
- Responsables de la medición
- Datos históricos y actuales del proceso

**Propósito:** Establecer la metodología de medición y datos baseline

---

### 02-GRAFICO_SECUENCIAL.xlsx
**Control Chart - Time Series Analysis**

Gráfico que muestra:
- Comportamiento del proceso en el tiempo
- Tendencias (subidas/bajadas)
- Puntos fuera de control
- Variación natural del proceso
- Ciclos o patrones

**Interpretación:**
- ¿Es el proceso estable?
- ¿Hay causas especiales afectando el proceso?
- ¿Qué variación es "normal"?

**Propósito:** Visualizar performance actual y variabilidad

---

### 03-GATE_REVIEW.xlsx
**Aprobación de Sistema de Medición**

Validación de:
- ✓ Datos de calidad aceptable
- ✓ Sistema de medición reproducible
- ✓ Suficientes datos recopilados
- ✓ Baseline establecido
- ✓ Autorización para análisis

---

## 📊 Métrica Clave Establecida

**Baseline Performance:**
- Defect rate actual
- Variación (desviación estándar)
- Tendencia del proceso

**Capability del Proceso (Antes):**
- Cp y Cpk values
- Sigma level
- Percentage of defects

---

## 🔬 Herramientas Estadísticas Utilizadas

- **Control Charts (I-MR):** Individual and Moving Range
- **Descriptive Statistics:** Mean, Std Dev, Min/Max
- **Normality Test:** Verificar distribución normal
- **Trend Analysis:** Patrones en el tiempo

---

## 📈 ¿Qué Aprendemos de Esta Fase?

1. **Magnitud del Problema:**
   - Cuántos defectos exactamente
   - Cuán variable es el proceso
   - Dónde está fuera de especificación

2. **Patrón de Variación:**
   - ¿Cambios repentinos?
   - ¿Tendencia gradual?
   - ¿Ciclos recurrentes?

3. **Capacidad del Proceso:**
   - ¿Puede el proceso cumplir especificaciones?
   - ¿Cuánta mejora es necesaria?

---

## ✅ Resultado de Esta Fase

Tenemos ahora:

1. **Datos confiables** del proceso actual
2. **Baseline cuantificado** para comparar mejoras
3. **Entendimiento visual** de la variación
4. **Baseline stats** (media, desv. estándar, capability)
5. **Autorización** para proceder al análisis de causas
